# 工程师助手 - 系统功能抽象总结

## 一、系统定位

AI驱动的企业级工程知识管理与协作平台，面向新能源车辆开发领域，通过**Web端**入口，实现技术文档的智能化管理、检索与生成。

---

## 二、核心功能抽象

### 1. 知识摄取层 (Knowledge Ingestion)

```
┌─────────────────────────────────────────────────────────┐
│                    知识摄取层                            │
├─────────────────────────────────────────────────────────┤
│  输入源                                                  │
│  ├── 本地文档 (PDF/Word/TXT)                            │
│  └── 飞书文档 (在线协作文档)                             │
├─────────────────────────────────────────────────────────┤
│  处理流程                                                │
│  文档上传 → 格式解析 → 文本提取 → 向量化 → 存储入库     │
├─────────────────────────────────────────────────────────┤
│  核心能力                                                │
│  • 多格式解析 (PyMuPDF + pdfplumber)                    │
│  • OCR识别 (DeepSeek-OCR)                              │
│  • 语义向量化 (Qwen3-Embedding)                         │
│  • 元数据提取 (标题/版本/页码)                          │
└─────────────────────────────────────────────────────────┘
```

**抽象定义**：将非结构化工程文档转化为可检索、可计算的向量知识。

---

### 2. 知识检索层 (Knowledge Retrieval)

```
┌─────────────────────────────────────────────────────────┐
│                    知识检索层                            │
├─────────────────────────────────────────────────────────┤
│  检索模式                                                │
│  ├── 向量语义检索 (Milvus)                              │
│  ├── 关键词检索                                         │
│  └── 混合检索 + 重排序 (Qwen3-Reranker)                │
├─────────────────────────────────────────────────────────┤
│  核心能力                                                │
│  • 语义理解 (自然语言查询)                              │
│  • 多轮对话上下文维护                                   │
│  • 引用溯源 (来源文档 + 页码)                           │
│  • 知识库范围限定                                       │
└─────────────────────────────────────────────────────────┘
```

**抽象定义**：基于用户意图，从知识库中精准定位相关技术信息。

---

### 3. 智能对话层 (Intelligent Conversation)

```
┌─────────────────────────────────────────────────────────┐
│                    智能对话层                            │
├─────────────────────────────────────────────────────────┤
│  对话引擎                                                │
│  ├── LangGraph Agent (状态管理 + 流程编排)              │
│  └── Qwen3-235B-A22B (大语言模型推理)                   │
├─────────────────────────────────────────────────────────┤
│  核心能力                                                │
│  • 自然语言理解与生成                                   │
│  • 多轮对话上下文记忆                                   │
│  • RAG增强生成 (检索增强)                               │
│  • 流式输出 (SSE)                                       │
└─────────────────────────────────────────────────────────┘
```

**抽象定义**：基于检索到的知识，生成准确、可溯源的技术回答。

---

### 4. 经验沉淀层 (Episode Management)

```
┌─────────────────────────────────────────────────────────┐
│                    经验沉淀层                            │
├─────────────────────────────────────────────────────────┤
│  Episode结构                                             │
│  ├── 问题描述 (Question)                                │
│  ├── 推理过程 (Reasoning)                               │
│  ├── 解决方案 (Solution)                                │
│  └── 引用来源 (References)                              │
├─────────────────────────────────────────────────────────┤
│  核心能力                                                │
│  • 从对话中提取结构化经验                               │
│  • Episode语义检索                                      │
│  • 经验复用与编辑                                       │
└─────────────────────────────────────────────────────────┘
```

**抽象定义**：将有价值的对话沉淀为可复用的工程经验资产。

---

### 5. 文档生成层 (Document Generation)

```
┌─────────────────────────────────────────────────────────┐
│                    文档生成层                            │
├─────────────────────────────────────────────────────────┤
│  生成要素                                                │
│  ├── 输入：参考文档 + 模板 + 提示词                     │
│  └── 输出：Markdown格式工程文档                         │
├─────────────────────────────────────────────────────────┤
│  内置模板                                                │
│  • FRS生成模板 (功能需求规范)                           │
│  • SSTS生成模板 (子系统技术规范)                        │
│  • 测试用例模板                                         │
│  • 技术评审模板                                         │
│  • 故障分析模板                                         │
├─────────────────────────────────────────────────────────┤
│  核心能力                                                │
│  • 多文档联合理解                                       │
│  • 模板化文档生成                                       │
│  • 在线编辑与导出                                       │
└─────────────────────────────────────────────────────────┘
```

**抽象定义**：基于模板和参考材料，自动生成标准化工程文档。

---

### 6. 知识管理层 (Knowledge Management)

```
┌─────────────────────────────────────────────────────────┐
│                    知识管理层                            │
├─────────────────────────────────────────────────────────┤
│  知识库类型                                              │

│  ├── 科室知识库 (部门共享)                              │
│  └── 中心知识库 (全员共享)                              │
├─────────────────────────────────────────────────────────┤
│  核心能力                                                │
│  • 知识库创建与分类                                     │
│  • 文档添加与移除                                       │
│  • 权限控制                                             │
│  • 标签管理                                             │
└─────────────────────────────────────────────────────────┘
```

**抽象定义**：组织和管理工程知识的容器，支持多级权限隔离。

## 五、核心价值总结

| 价值维度 | 实现方式 |
|---------|---------|
| **信息聚合** | 多源文档统一入库，向量化检索 |
| **效率提升** | 自然语言交互，秒级响应 |
| **经验传承** | Episode机制沉淀工程经验 |
| **智能生成** | 模板化自动生成工程文档 |
| **团队协作** | 飞书群聊无缝集成 |
